## ScraperAPI 到底是什么

简单讲：它是一个把代理轮换、浏览器指纹伪装、验证码处理、请求重试全部打包成一个 API 端点的服务。你不用自己维护代理池，不用写轮换逻辑，甚至不用操心 User-Agent 和 Cookie 管理。发一个 HTTP 请求过去，它在后端帮你搞定这些脏活，返回干净的 HTML。

对做爬虫的人来说，这意味着你只需要关心"解析数据"这一步，前面的反爬对抗全部外包。

## 常见问题

### ScraperAPI 的代理轮换机制和自建代理池有什么区别？

自建代理池你得自己买 IP、写轮换逻辑、处理封禁检测和 IP 回收。ScraperAPI 把这些全部封装了，你只管发请求，它在后端用算法自动选 IP、自动重试失败请求、自动绕验证码。维护成本从"一个人全职盯"降到"零"。

### 抓取 JavaScript 渲染的页面支持吗？

支持。请求参数加上 `render=true`，ScraperAPI 会用无头浏览器加载页面，等 JS 执行完再返回完整 DOM。我用它抓过 React 写的 SPA 站点，没问题。
